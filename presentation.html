Ричард Докинз, «Слепой часовщик»

Эволюция биоморф

Генетический алгоритм на языке Haskell

геном → фенотип → отбор по сходству → мутация → повтор

Идея

Программа имитирует кумулятивный отбор — основную мысль книги Докинза:

  • нужная форма не появляется за один случайный геном;
  • она постепенно приближается к целевой картинке за счёт накопления мелких наследуемых изменений под действием отбора.

Почему «слепой» часовщик

У эволюции нет цели и нет зрения. Роль «зрения» здесь играет автоматическое сравнение с заданной картинкой (котик).

Что делаем по шагам

  1. генерируем случайный геном;
  2. создаём N = 20 потомков-мутантов;
  3. строим биоморфу для каждого;
  4. сравниваем с целью (фитнес);
  5. лучший становится родителем следующего поколения;
  6. стоп, если 10 поколений нет улучшения.

Общая схема алгоритма

Генотип: 16 генов

Гены 1–15 — скелет

значения −9..+9, кодируют положение каждого отрезка.

Ген 16 — длина

значения 2..12, задаёт длину отрезков биоморфы.

-- Genetic.hs: случайный геном = 15 скелетных + 1 длина
randomGenotype generator =
    let (skeleton, g1) = generateGenes generator 15 (-9, 9)
        (len, g2)      = randomR (2, 12) g1
    in (skeleton ++ [len], g2)

Выбор 7 генов из 15

Для построения скелета используются 7 случайно выбранных позиций. Они выбираются один раз в начале эволюции (в модуле Evolution) и далее фиксированы для всех поколений.

-- 7 позиций из [0..14], выбираются один раз
getRandomPositions gen 0 _     = ([], gen)
getRandomPositions gen n range =
  let (pos,  gen')  = randomR range gen
      (rest, gen'') = getRandomPositions gen' (n-1) range
  in (pos:rest, gen'')

-- по этим позициям берём гены любого генотипа
selectGenesByPositions positions genotype =
  map (genotype !!) positions

Почему один раз?

Если выбирать 7 генов заново на каждом шаге, один и тот же геном давал бы разную картинку, и фитнес стал бы непостоянным — отбор перестал бы работать.

Фиксированные позиции: геном всегда даёт одну и ту же биоморфу, отбор корректен.

Генотип → фенотип: строим скелет

Каждый ген превращается в направление (одно из восьми), и из центра холста рисуется цепочка отрезков.

-- ген → направление
genToDirections gene =
  toEnum ((gene + 9) `mod` 8)

-- рекурсивно идём по направлениям
buildVector start []           _   = []
buildVector start (dir:dirs) len =
  let (dx,dy) = directionVector dir
      next   = (fst start + dx*len,
                snd start + dy*len)
  in (start,next) : buildVector next dirs len

Скелет строится из 7 выбранных генов; длина шага задана константой stepLen = 10 (16-й ген в текущей версии в отрисовке не участвует).

Билатеральная симметрия

Скелет отражается относительно вертикальной оси холста — получается симметричная фигура.

-- отражение по X: x → size-1-x
buildMirrorVectors size segs = map mir segs
  where mir ((x1,y1),(x2,y2)) =
          ((size-1-x1, y1),
           (size-1-x2, y2))

allSegments = segs ++ buildMirrorVectors 150 segs

Рисуем segs и их зеркало на одном холсте.

Мутация: один ген на ±1

В каждом поколении в копии генома меняется ровно один случайный ген на единицу; новое значение остаётся в допустимых границах.

mutateGenotype g genotype =
  let (pos, g1)     = randomR (0, 15) g                  -- какой ген
      (newGene, g2) = mutateGen g1 (genotype !! pos) pos  -- ±1, не выходя за границы
  in (replaceAt pos newGene genotype, g2)

Фитнес: насколько похоже на цель

Мера непохожести — евклидово расстояние между пикселями без корня (корень не меняет порядок, а считать дешевле):

d² = Σ ( ai − bi )²

Чем меньше d², тем биоморфа похожее на котика. Отбор ищет минимум.

euclideanDistanceSquared img1 img2 =
    let pixels1 = toList img1
        pixels2 = toList img2
        diffSquared p1 p2 =
          let diff = fromIntegral p1 - fromIntegral p2
          in diff * diff
        sumSquares = sum (zipWith diffSquared pixels1 pixels2)
    in fromIntegral sumSquares

Наследование и отбор

Наследует ли новое поколение признаки предыдущего? Откуда берутся 20 потомков?

Да, наследует: 20 потомков — это мутации лучшего ребёнка предыдущего поколения. Родителем следующего поколения становится полный генотип лучшего (bestFullGen).
Глобально лучший отдельно не хранится: в Best_last.png попадает лучший за последнее поколение, а «улучшение» сравнивается с предыдущим поколением (нет «элитизма», возможны откаты).

Критерий остановки

Эволюция останавливается, когда сходство не улучшается 10 поколений подряд (стагнация).

  • сравниваем лучший фитнес поколения с лучшим из предыдущего поколения;
  • есть улучшение — счётчик стагнации обнуляется;
  • нет — счётчик увеличивается на 1;
  • счётчик ≥ 10 — стоп, сохраняем лучший за последнее поколение в Best_last.png.

Дополнительная страховка — лимит в 300 поколений.

loop gen n parent bestSel bf stagn
  | stagn >= 10 = save bestSel bf   -- лучший за последнее поколение
  | n > 300     = save bestSel bf   -- лимит поколений
  | otherwise = do
      let (pop, gen') = generatePopulation gen parent 20
      res <- processingChildrenWithSave positions target n pop
      let (bestSel', bestFull, bf') =
            minimumBy (\(_,_,a) (_,_,b) -> compare a b) res
          improved = bf' < bf * 0.999   -- сравнение с прошлым поколением
          stagn'   = if improved then 0 else stagn+1
      loop gen' (n+1) bestFull bestSel' bf' stagn'

Функциональный стиль (1/2)

1. Чистота и явная передача генератора

Случайность чистая: генератор протаскивается вручную g → g' → g''.

generateGenes g 0 _ = ([], g)
generateGenes g n r =
  let (v,  g')  = randomR r g
      (vs, g'') = generateGenes g' (n-1) r
  in (v:vs, g'')

2. Рекурсия по списку

getRandomPositions набирает 7 позиций, generateGenes — 15 генов: список собирается рекурсивно.

3. Сопоставление с образцом и охранники

directionVector N = ( 0,-1)
directionVector E = ( 1, 0)
-- ...
buildVector _ [] _ = []   -- базовый случай

loop … stagn
  | stagn >= 10 = …       -- охранники
  | otherwise  = …
